# GO-TONGX 开发规范

## 🔴 最高准则：模块化

> **所有功能必须模块化。** 每个功能做成独立模块/组件，不混入其他模块。

### 规则

1. **前端**：每个功能面板是独立的 `.tsx` 组件文件，自包含状态和逻辑
2. **后端**：每个功能域是独立的 `.rs` 模块文件，通过 `mod` 引入
3. **数据**：每个功能有独立的数据库表，不共用字段
4. **接口**：模块间通过 Tauri 命令或 props 通信，不共享内部状态
5. **可拆卸**：删除一个模块不应影响其他模块的正常运行

### 示例

```
✅ 正确: src/components/TokenMonitor.tsx (独立组件)
❌ 错误: 把 token 监控逻辑直接写在 SettingsManager.tsx 里

✅ 正确: src-tauri/src/token_usage.rs (独立模块)  
❌ 错误: 把 token 记录逻辑直接写在 ai.rs 里
```
